# Codebase Conventions

**Analysis Date:** 2026-04-27

## Naming Patterns

### Files
- Source: `source/component_name/component_name.cpp` / `source/component_name/component_name.h` — module name doubles as directory and filename stem. Examples: `source/encode/encode.cpp`, `source/transport/rist_transport.cpp`, `source/ndi_input/ndi_input.h`.
- Headers only: `include/common.h` — shared enums and structs across modules.
- All headers use `#pragma once` exclusively — no `#ifndef`/`#define` include guards anywhere in the project.
- Exception: URL module uses `.cc` extension (`source/url/url.cc`, `source/url/url.h`), likely to disambiguate from CMake's `.cc` language detection.
- FLTK-generated output: `source/ui/ui.cxx` (generated by FLUID), `source/ui/ui.fld` (FLUID designer XML).
- Test: `test/source/<name>_test.cpp` — test file mirrors source structure.

### Functions
- `lower_case` with underscore separators — enforced by clang-tidy `readability-identifier-naming.FunctionCase: lower_case` and all 47 naming entries in `.clang-tidy`. Example: `pipeline_build_video_encoder()`, `stats_thread_func()`, `refresh_devices()`.

### Variables
- Member variables: `m_` prefix with `lower_case` — prefix and naming enforced by `.clang-tidy` (`PrivateMemberPrefix: 'm_'`). Examples: `m_running`, `m_connected`, `m_stats_mutex`, `m_log_callback`.
- Local variables: `lower_case`, no prefix. Examples: `pipeline_string`, `interface_list`.
- Global state: lowercase, no prefix — `library app;`, `encode* ptr_encoder;` in `source/main.cpp`.

### Types
- Classes, structs, enums, and enum constants: all `lower_case` — enforced by clang-tidy. Examples: `class encode`, `struct stream_config`, `enum class codec`, `codec::h264`.
- Template parameters: `CamelCase` — enforced by `.clang-tidy` (`TemplateParameterCase: 'CamelCase'`, `TypeTemplateParameterCase: 'CamelCase'`).
- Type aliases: `lower_case` — e.g., `using FuncPtr = void (*)();`, `using transport_ptr = std::shared_ptr<transport_base>;`.

## Code Style

### Formatting
- Tool: `clang-format` (v14 preferred, configurable via `FORMAT_COMMAND` CMake variable).
- Run: `cmake --build build -t format-fix` (developer mode).
- CI: `cmake -D FORMAT_COMMAND=clang-format-14 -P cmake/lint.cmake` — runs as `format-check` target, fails on violations.
- Key settings from `.clang-format`:
  - 80-column limit (`ColumnLimit: 80`)
  - 2-space indent (`IndentWidth: 2`)
  - Left pointer alignment (`PointerAlignment: Left`) — write `type* ptr`, not `type *ptr`
  - `BreakBeforeBraces: Custom` — Allman-style: braces on their own line for functions, classes, namespaces
  - `AllowShortFunctionsOnASingleLine: Inline` — short single-line functions OK
  - `AllowShortIfStatementsOnASingleLine: Never` — always use braces for if
  - `SpacesInAngles: false` — no spaces inside `<` `>` — write `std::vector<int>`, not `std::vector< int >`
  - `IncludeBlocks: Regroup` + ordered categories: `<...>` (system) → `*.h` → `<...>` → `"..."` (project)

### Linting
- Tool: `clang-tidy-14`, enabled via `clang-tidy` CMake preset (`CMAKE_CXX_CLANG_TIDY`).
- Config: `.clang-tidy` — enables `*` (all checks) with specific exclusions:
  - Excluded: `google-readability-todo`, `altera-*`, `fuchsia-*`, `llvm-header-guard`, `llvm-include-order`, `llvmlibc-*`, `modernize-use-nodiscard`, `misc-non-private-member-variables-in-classes`
  - Key rules: `bugprone-sizeof-expression.WarnOnSizeOfIntegerExpression`, `readability-else-after-return`, `readability-qualified-auto`, `cppcoreguidelines-narrowing-conversions`
- Spell checking: `codespell` via `cmake/spell.cmake`, skips `build/` and `prefix/`.

## Import Organization

Order (enforced by clang-format `IncludeCategories`):
1. Standard library headers: `#include <cstddef>`, `#include <cstdint>`, `#include <string>`, etc.
2. Third-party headers: GStreamer (`<gst/gst.h>`), FLTK (`<FL/Fl.H>`), NDI, rist
3. Project headers: `"lib.h"`, `"encode.h"`, `"transport.h"`, etc. (quoted includes)

- `#include "FL/fl_callback_macros.H"` — note capital `.H` for FLTK callback macros header.
- Never use `using namespace std;` — the project avoids this. Selective `using std::string;` appears in a handful of `.cpp` files (`transport.cpp`, `rist_transport.cpp`, `encode.cpp`) to reduce verbosity.
- Header guard: `#pragma once` exclusively — no `#ifndef` guards used anywhere.

## Error Handling

- Constructor failure: `try`/`catch` with `std::terminate()` for atomic member init exceptions. Example: `source/lib/lib.cpp:3-7`.
- GStreamer errors: parsed via `gst_message_parse_error()` with structured log output, including element name and debug info. Example: `source/main.cpp:156-176`, `source/ndi_input/ndi_input.cpp:95-112`.
- `std::format` for all error message formatting — zero use of `sprintf`, `std::to_string` concatenation, or `std::stringstream`.
- Transport operations: non-blocking with callbacks — `send_buffer()` calls `callback(false, "error string")` on failure. No exceptions in transport layer.
- No custom exception types — the project relies on GStreamer error messages and callback-based error propagation.

## Logging

- Pattern: callback function pointer stored as member, invoked via `log(msg)` method. Example: `encode::log()` in `source/encode/encode.cpp:519`, `ndi_input::log()` in `source/ndi_input/ndi_input.cpp:129`.
- `transport_base` uses `std::function<void(const std::string& msg)>` for callbacks — more modern than raw function pointers.
- UI displays: `encode_log_append()` and `transport_log_append()` append to `Fl_Text_Buffer` members for display in `Fl_Text_Display` widgets.
- Logging always includes trailing `\n` in messages.
- GStreamer debug: `GST_DEBUG` env vars for runtime pipeline debugging (`GST_DEBUG=3`, `GST_DEBUG_DUMP_DOT_DIR=/tmp`).

## Module / Function Design

### Function patterns
- Constructor with `try`/`catch` for atomic member initialization — `library::library() noexcept try : member {value} {} catch (...) { std::terminate(); }`. Seen in `source/lib/lib.cpp`.
- `explicit` constructors: all non-default constructors that take a single argument are `explicit`. Examples: `explicit encode(...)` (`source/encode/encode.h:27`), `explicit ndi_input(...)` (`source/ndi_input/ndi_input.h:15`), `explicit add_output_dialog(...)` (`source/ui/add_output_dialog.h:24`).
- Pipeline builder method decomposition: `encode` class has 20+ private `pipeline_build_*()` methods, each handling one encoder variant. No single function exceeds 50 lines.
- Switch-on-enum pattern: heavy use of `switch` statements for enum dispatch — 17 switch blocks in `encode.cpp` alone, handling encoder+codec combinations. All switches are exhaustive (no `default` case needed — enums are small and complete).

### Parameter patterns
- Config structs passed by `const&`: `const input_config&`, `const encode_config&`, `const stream_config&`.
- Raw pointer parameters for GStreamer: `GstMessage* message`, `GError* err` — GStreamer's C API convention.
- Callback function pointers: `void (*)(const std::string&)` for logging, `std::function<...>` for callbacks.

### Return values
- Boolean success: `bool initialize(...)`, `bool start()`, `bool is_connected()` — all transport operations return bool.
- `nullptr` for lookup failures: `get_transport()` returns `nullptr` if not found.
- Struct returns: `stream_stats`, `buffer_data` — returned by value (structs are small).

### Exports
- All public symbols are in `source/` — no exported header in `include/` (the `include/` dir only contains `common.h` with shared enums).
- CMake: targets named `open-broadcast-encoder_exe` and `open-broadcast-encoder_lib` with ALIAS.

## Convention Adoption Rates

- **All-identifier `lower_case`:** `~95% (stable)` — 47 clang-tidy rules enforce this across all identifier categories; zero `CamelCase` violations found in production source.
- **`m_` prefix on member variables:** `~75% (rising)` — 116 occurrences in source headers; newer transport/UI files (transport_manager, buffer_distributor, rist_transport, stats_aggregator) consistently use it; older files (encode.h, lib.h) omit it.
- **`#pragma once` over `#ifndef` guards:** `~100% (stable)` — only header in project (`include/common.h`) uses it; no `#ifndef` guards found.
- **`std::format` for string formatting:** `~100% (stable)` — 33 occurrences in source `.cpp` files, zero `sprintf`/`snprintf`/`stringstream` usage.
- **Explicit constructors:** `~85% (stable)` — 3 single-argument constructors use `explicit` (encode, ndi_input, add_output_dialog). The one that doesn't is `transport()` (default-only).
- **80-column line limit:** `~90% (stable)` — enforced by clang-format `ColumnLimit: 80`; violations caught by CI `format-check` job.
- **Left pointer alignment (`type* ptr`):** `~95% (stable)` — clang-format `PointerAlignment: Left`; `GstElement* bus = nullptr;` in `encode.h` confirms.
- **Callback-based error propagation:** `~60% (rising)` — `transport_base` uses `std::function<...>` callbacks for errors; legacy `transport` class uses C-style function pointers.

## Testing And Mocking

### Test types used
- **Unit:** Yes — Catch2 v3 with `Catch2WithMain`.
- **Integration:** Not detected.
- **E2E:** Not detected.

### Where tests live
- `test/source/open-broadcast-encoder_test.cpp` — the single test file.

### Test framework
- Catch2 v3: `#include <catch2/catch_test_macros.hpp>`, `TEST_CASE("...", "[tag]")`, `REQUIRE(...)`.
- CMake: `find_package(Catch2 REQUIRED)`, `include(Catch)`, `catch_discover_tests()`.

### Current test coverage
- Only 1 test: verifies `library{}.name == "open-broadcast-encoder"`.
- The test file is minimal — no mocks, no factories, no fixtures.

### Mocking boundaries (explicit)

- **Do mock:** External hardware/OS interfaces — `RISTNetSender` (hardware network protocol), NDI device enumeration (requires NDI runtime), GStreamer pipeline elements (requires GStreamer runtime with all plugins).
  - Example: When testing `rist_transport::send_buffer()`, mock the underlying `m_sender->sendData()` call rather than running real network traffic.

- **Do NOT mock:** `buffer_distributor` core logic, `transport_manager::create_transport()` factory, callback registration. These are pure C++ logic — mocking them defeats the purpose of integration testing.
  - Example: `buffer_distributor::distribute()` should be tested with real `transport_base` stubs that record calls, not with mocked `std::function` callbacks.

### CI reliability rules
- **Lint must pass before test:** CI jobs declare `needs: [lint]` — `coverage`, `sanitize`, and `test` jobs all depend on the `lint` job completing first.
- **Full matrix:** Tests run on `macos-14`, `ubuntu-22.04`, `windows-2022` simultaneously.
- **Sanitizer job:** ASan + UBSan runs with `halt_on_error=1` — any detected undefined behavior fails CI.
- **Format check runs first:** `cmake -D FORMAT_COMMAND=clang-format-14 -P cmake/lint.cmake` fails the build if any file is not formatted.
- **No test parallelism constraints:** Tests run with `-j 2` in CI.
- **Test timeouts:** Not explicitly set — relies on Catch2 defaults (infinite for this codebase, since it's a real-time streaming application).

## External Integration Patterns

### GStreamer Pipeline Building
- Pipelines built as string templates via `std::format`, passed to `gst_parse_launch()`.
- Elements named with `name=...` for later retrieval: `gst_bin_get_by_name(GST_BIN(pipeline), "videncoder")` in `source/encode/encode.cpp:386`.
- Pattern: build pipeline string → `gst_parse_launch()` → `gst_element_set_state()` → manage via `GstBus` message loop.
- Resource cleanup: `gst_element_set_state(GST_STATE_NULL)` + `gst_object_unref()` in destructors.

### FLTK Threading (Critical)
- All UI updates from background threads MUST use this exact pattern:
  ```cpp
  Fl::lock();       // NOT ui.lock() — uses global FLTK lock
  // ... update UI ...
  Fl::unlock();
  Fl::awake();      // Required to wake the FLTK event loop
  ```
- See `source/ui/ui.cpp:487-594` for repeated instances. The `Fl::lock()` check pattern — `if (Fl::lock()) { ... Fl::unlock(); Fl::awake(); }` — is used for thread-safe UI reads.

### FLTK Callbacks
- Use `FL_METHOD_CALLBACK_N` macros from `FL/fl_callback_macros.H`.
- Menu items store enum values as `user_data()` — cast via `static_cast<long>` (set) and `reinterpret_cast<uintptr_t>` (retrieve).

### Transport Architecture
- Abstract base `transport_base` with three concrete implementations: `rist_transport`, `srt_transport`, `rtmp_transport`.
- `transport_manager::create_transport()` is the factory — switch on `transport_protocol` enum.
- `buffer_distributor` owns the single → multi output fan-out using `shared_ptr` for zero-copy distribution.

### Global State
- `source/main.cpp` declares four global variables: `library app`, `std::unique_ptr<transport> transporter`, `user_interface ui`, `encode* ptr_encoder`.
- Cross-component communication goes through these globals — not through dependency injection.
- The `library` struct holds all config objects (`input_config`, `encode_config`, `output_config`, `stream_config`).

## Golden Files

These files have the highest density of documented conventions — they are production code (not scaffolding, not generated, not tests).

- `source/transport/transport_base.h` — exemplifies `lower_case` class naming, `m_` prefix on protected members (`m_stats_callback`, `m_log_callback`, `m_config`, `m_stats`), `std::function` callback typedefs, `#pragma once`, virtual method patterns (`initialize()`, `start()`, `stop()`, `send_buffer()`), and Doxygen `@brief`/`@details` documentation blocks.
- `source/transport/buffer_distributor.h` — exemplifies modern `m_` prefix on all private members (`m_transports`, `m_mutex`, `m_connection_callback`, `m_running`), `using` type aliases (`transport_ptr`, `connection_callback`), callback pattern with `std::function`, mutex-protected access (`mutable std::mutex m_mutex`), `std::shared_ptr` zero-copy buffer distribution, and constructor with `std::atomic<bool>{false}` inline initialization.
- `source/main.cpp` — exemplifies global state hub pattern (`library app`, `encode* ptr_encoder`), static callback forwarding (`encode_log()`, `transport_log()`), `std::format` usage, `switch`-on-enum dispatch, `std::thread` with detach for preview, `Fl::lock()`/`Fl::unlock()`/`Fl::awake()` for UI thread safety, and `explicit` constructor usage via factory pattern.
